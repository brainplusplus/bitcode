# Fix 1-12: Consolidate CLI Entry Points

**Date**: 2026-04-25
**Status**: Done

## Summary

Merged `cmd/engine` into `cmd/bitcode` as a single unified CLI binary. Replaced `go build` workflow with `go install` in all run scripts.

## Changes

### Consolidated CLI (`cmd/bitcode`)

- Added `bitcode serve` command (replaces `cmd/engine/main.go`)
- Improved `bitcode dev` with auto-detection:
  - **Engine repo mode**: detects `go.mod` with `github.com/bitcode-framework/bitcode`, uses Air CLI flags (no `.air.toml` needed)
  - **App project mode**: watches module files (JSON, HTML, templates), reloads in-process
  - Override flags: `--engine` / `--no-engine`
- Removed `cmd/engine/` directory entirely
- Removed `samples/erp/.air.toml` (Air now invoked via CLI flags)

### Run Scripts

- `run.bat` ‚Äî updated: `go install` ‚Üí `bitcode dev`
- `run.ps1` ‚Äî updated: `go install` ‚Üí `bitcode dev`
- `run.sh` ‚Äî **new**: Linux/macOS support

### Makefile

- `make build` ‚Üí builds single `bin/bitcode` binary
- `make install` ‚Üí `go install ./cmd/bitcode/`
- `make serve` ‚Üí `go run ./cmd/bitcode/ serve`
- `make dev` ‚Üí `go run ./cmd/bitcode/ dev`
- Removed `make cli` (redundant, `build` now builds the CLI)

### Docs Updated

- `README.md` ‚Äî Quick Start, CLI commands, PostgreSQL/MySQL examples
- `AGENTS.md` ‚Äî file structure, build section, test count
- `docs/codebase.md` ‚Äî cmd/ tree, samples section, build & run section
- `samples/erp/README.md` ‚Äî Quick Start, multi-tenancy, PostgreSQL, Redis examples
- `engine/docs/features/multitenancy.md` ‚Äî run command

## Rationale

1. **Name collision**: `engine` is too generic, conflicts with other tools in PATH
2. **Single entry point**: one binary (`bitcode`), like Laravel's `artisan` or Rails' `bin/rails`
3. **DRY**: `cmd/engine` was functionally identical to `bitcode serve`
4. **`go install` over `go build`**: binary goes to `$GOPATH/bin` (already in PATH), usable from anywhere
5. **No `.air.toml`**: Air supports full CLI flags, keeps project directory clean
